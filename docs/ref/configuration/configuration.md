# Configuration

This project uses two main configuration files, both located in `group_vars/all/`:

## Configuration Files

### group_vars/all/main.yml

The primary configuration file containing all deployment variables. This file is generated by the setup wizard (`setup.sh` or `setup-tui.sh`) and controls:

- **Wazuh version** and component settings
- **Node definitions** (indexer, manager, dashboard IPs and names)
- **Feature toggles** (vulnerability detection, FIM, SCA, rootkit detection, etc.)
- **Integration settings** (email alerts, syslog output, Slack, VirusTotal, cloud providers)
- **Cluster configuration** (manager clustering, indexer cluster settings)
- **Deployment behavior** (lockdown settings, firewall configuration, monitoring)

Edit this file directly to change deployment configuration:

```bash
# Edit configuration
vi group_vars/all/main.yml

# Or use make
make vault-edit  # for vault only
```

### group_vars/all/vault.yml

An Ansible Vault-encrypted file containing all sensitive credentials:

- Wazuh API passwords
- Indexer admin credentials
- Dashboard credentials
- Agent enrollment passwords
- Cluster keys

This file is encrypted at rest using the password stored in `.vault_password`. Manage it with:

```bash
# View credentials
./scripts/manage-vault.sh view

# Edit credentials
./scripts/manage-vault.sh edit

# Rotate all passwords
./scripts/manage-vault.sh rotate

# Change the vault encryption password
./scripts/manage-vault.sh rekey
```

## How Vault Password Is Loaded

The generated `ansible.cfg` includes `vault_password_file = .vault_password`, so you do not need to pass `--vault-password-file` on every command. Ansible reads the vault password automatically.

## Other Configuration Files

| File | Purpose |
|------|---------|
| `inventory/hosts.yml` | Ansible inventory defining which hosts get which roles |
| `inventory/bootstrap.yml` | Bootstrap inventory (connects as root for initial setup) |
| `ansible.cfg` | Ansible settings (SSH key path, vault password file, etc.) |

All of these files are generated by the setup wizard and typically do not need manual editing.
